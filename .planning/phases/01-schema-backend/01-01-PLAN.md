## Decisions to implement

- **D-04 (Error contract):** `insert_batch()` and `query()` return `InsertResult`/`QueryResult` with `.success`, `.rows_inserted`/`.rows`, `.error`. No exceptions for operational errors.
- **D-02 (Schema versioning):** Lightweight — `schema_meta` table with single `version` integer. `CREATE TABLE IF NOT EXISTS` for idempotency.
- **D-03 (ClickHouse table layout):** `events` table with `ORDER BY (timestamp, instrument_id, event_type)`, daily partitions.
- **D-01 (Payload encoding):** msgpack bytes stored as ClickHouse `String`. The backend interface accepts rows with pre-serialized payload strings.

## DDL (from CONTEXT.md specifics)

```sql
CREATE TABLE IF NOT EXISTS events (
    timestamp DateTime64(3),
    event_type LowCardinality(String),
    instrument_id LowCardinality(String),
    module_id String,
    payload String
) ENGINE = MergeTree()
PARTITION BY toYYYYMMDD(timestamp)
ORDER BY (timestamp, instrument_id, event_type)
```

```sql
CREATE TABLE IF NOT EXISTS schema_meta (
    version UInt32,
    applied_at DateTime64(3)
) ENGINE = MergeTree()
ORDER BY version
```
